03 / First machine

Start with a draft.
Keep it small. Draft에서 시작하세요.
작게 유지하세요.

DraftStateMachine.kt 
sealed interface DraftPhase {
    data object Editing : DraftPhase
    data object Saving : DraftPhase
    data object Saved : DraftPhase
}

data class DraftData(
    val title: String = "",
    val errorMessage: String? = null,
)

typealias DraftState = AfsmState<DraftPhase, DraftData>

sealed interface DraftEvent {
    data class TitleChanged(val value: String) : DraftEvent
    data object SaveClicked : DraftEvent
    data object SaveCompleted : DraftEvent
}

sealed interface DraftCommand {
    data class Save(val title: String) : DraftCommand
}

val draftMachine = afsmMachine<
    DraftPhase, DraftData, DraftEvent, DraftCommand
> {
    initial(DraftPhase.Editing, DraftData())

    phase(DraftPhase.Editing) {
        on<DraftEvent.TitleChanged> {
            updateData { data, event ->
                data.copy(title = event.value)
            }
        }

        on<DraftEvent.SaveClicked> {
            case("valid title", condition = { data.title.isNotBlank() }) {
                transitionTo(DraftPhase.Saving)
            }
            case("missing title", condition = { data.title.isBlank() }) {
                updateData { copy(errorMessage = "Title is required.") }
            }
        }
    }

    phase(DraftPhase.Saving) {
        onEnter {
            command("Save") { DraftCommand.Save(data.title) }
        }
        on<DraftEvent.SaveCompleted> {
            transitionTo(DraftPhase.Saved)
        }
    }

    phase(DraftPhase.Saved)
}

Read this machine from the outside in. First name the product flow, then make each phase’s valid behavior local. 머신을 바깥에서 안쪽으로 읽으세요. 먼저 제품 흐름에 이름을 붙이고, 각 phase에서 유효한 동작을 가까이에 둡니다.

1. Name the flow 1. 흐름에 이름 붙이기

Phase says where you are. Data says what must survive while you are there. Phase는 현재 단계를, Data는 그 단계에서 유지해야 할 값을 말합니다.

2. Keep rules phase-local 2. 규칙을 phase 가까이에 두기

An event handler exists only where that event is meaningful. Missing handlers are invalid by default. event handler는 해당 event가 의미 있는 phase에만 둡니다. 누락된 handler는 기본적으로 invalid입니다.

3. Emit work; do not perform it 3. 작업은 실행하지 말고 요청하기

Entering Saving emits a Command. ViewModel executes it and returns SaveCompleted. Saving에 진입하면 Command를 냅니다. ViewModel이 실행하고 SaveCompleted를 돌려줍니다.

    06 / Fit

    Use the machine
    only when it earns its place.     머신이 가치를 만들 때만
    사용하세요.

    Good fit 잘 맞는 화면

    • Multi-step business flows여러 단계의 비즈니스 흐름
    • Retries and stale async results재시도와 오래된 비동기 결과
    • Rules that depend on the current phase현재 phase에 따라 달라지는 규칙
    • Flows that need reviewable topology검토 가능한 topology가 필요한 흐름

    Keep it ordinary 일반 방식 유지

    • Simple loading / content / error screens단순한 loading / content / error 화면
    • UI-only focus, scroll, sheet, and animation stateUI 전용 focus, scroll, sheet, animation state
    • A flow already clearer as ViewModel + StateFlowViewModel + StateFlow가 이미 더 명확한 흐름
    • Anything that would add ceremony without safety안전성 없이 의식적인 코드만 늘어나는 경우
