## Findings and Corrections

### Current state was a chronological implementation log

The page accumulated hundreds of dated implementation details, intermediate
spikes, review conclusions, and superseded vocabulary. It also began by saying
the project was preparing to build a library even though the full internal-beta
surface exists.

Correction: rewrite it as a concise current synthesis organized by product
position, architecture contract, implemented modules, runtime policy, examples,
verification, distribution, and remaining decisions. Historical detail remains
in decision, implementation, meeting, and wiki logs.

### Open questions mixed resolved policy with real decisions

The page still asked about effect ownership, retry ownership, invalid transition
defaults, graph labels, graph branch representation, restoration behavior,
audience, current distribution, and sample selection even though code and public
docs answer those questions.

Correction: retain only public release identity/compatibility, final API/module
boundaries, advanced runtime scope, restoration helper scope, and graph
aggregation. Keep a short resolved-policy section so future agents do not reopen
settled behavior.

### Hosted CI policy contradicted the repository

`current-state.md`, `CHANGELOG.md`, `CONTRIBUTING.md`, and one release-readiness
gate still described an active GitHub Actions workflow after that workflow had
been removed for cost control.

Correction: make the local release script authoritative everywhere, record the
hosted workflow as removed, and state that any future CI must call the same
script rather than duplicate it.

### Historical pages were presented as current guidance

The wiki index described the v1/v2 public API drafts and early runtime/delivery
plans without marking them historical. The v2 page still called itself the
design source despite using removed API names.

Correction: add explicit historical status notices and update index labels.

### Index coverage missed the append-only wiki log

All content pages linked from the index resolved, but `wiki/log.md` had no index
entry.

Correction: add the Wiki Log and this lint report to `wiki/index.md`.

## Remaining Gaps

- Older engineering and meeting pages intentionally preserve superseded terms.
  A future terminology/provenance lint should verify that every historical page
  is clearly labeled and that no current public doc links to obsolete API names.
- The unresolved decisions in `open-questions.md` require product or public API
  choices; current code alone cannot close them.
- Android CLI evidence under `raw/verification/` is dated historical evidence.
  A new device smoke run is required before making a current UI journey claim.
