# Wiki Lint 2026-07-17

## Scope

Compare the Effect-free core/runtime/ViewModel/sample implementation with the
canonical context, product, Android boundary, restoration, testing, experiment,
index, and first-use pages. Check that removed four-channel APIs cannot be
mistaken for the current contract while preserving dated evidence.

## Code Evidence

- `AfsmReducer<S, E, C>` and `AfsmMachine<S, E, C>` have three type channels.
- `AfsmDefaultMachine<S, E, C>` adds a genuine default state.
- `AfsmTransition<S, C>` has state, commands, command invocations, and decision;
  it has no Effect output.
- `AfsmHost` owns serialized Event processing, StateFlow publication, and
  command execution.
- sample screens construct zero Auth, Checkout, or ProductEditor Event values;
  sample ViewModels expose feature verbs.

## Findings And Corrections

### Current Android boundary was stale

`viewmodel-fsm-boundaries.md` still told UI to construct Events, ViewModel to
expose `onEvent`, and navigation to follow a state/effect policy.

Correction: document verb-named UI methods, ViewModel-internal Event
translation, `AfsmHost` ownership, durable completion, and direct/state-driven
UI behavior.

### Current restoration and testing pages retained Effect wording

The Checkout restoration experiment still referred to replaying
`PaymentCompleted` and emitting an Effect after restoration. The testing
strategy still prescribed event callbacks and state/effect UI policy.

Correction: restored completion is durable state with no command or separate
one-shot output; tests cover feature verbs, internal result Events, decisions,
stale results, invocations, restoration, and direct/state-driven UI behavior.

### Dynamic-initial-state page used the removed arity

Its selected machine/default-machine split is still current, but the sample
interfaces retained the old fourth Effect type.

Correction: keep the page current and update it to the implemented
three-generic contracts.

### Legacy experiments looked current when opened directly

The old first-use type-channel experiments, Signup reference/pseudo code, v2
compile validation, and Checkout Effect integration verification were useful
history but lacked consistent page-head routing.

Correction: add Historical titles/statuses and current-contract links, and
label their index entries as historical without rewriting the original
evidence.
